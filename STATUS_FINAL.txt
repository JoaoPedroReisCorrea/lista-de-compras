🔧 ROTAS DA API IMPLEMENTADAS
═══════════════════════════════════════════════════════════════════════════

GET /api/produtos
├─ Descrição: Retorna todos os produtos cadastrados
├─ Método: GET
├─ Body: (nenhum)
├─ Resposta: { sucesso, dados[], total }
└─ Status: 200 OK / 500 ERROR

POST /api/produtos
├─ Descrição: Insere novo produto no banco
├─ Método: POST
├─ Body: { nome, categoria, descricao }
├─ Resposta: { sucesso, mensagem, id }
├─ Status: 201 CREATED / 400 BAD REQUEST / 500 ERROR
└─ Validação: nome e categoria obrigatórios

═══════════════════════════════════════════════════════════════════════════
📞 SUPORTE RÁPIDO
═══════════════════════════════════════════════════════════════════════════

Problema: Dados não aparecem na listagem
Solução:
  1. Verifique se backend está rodando (npm start)
  2. Abra DevTools (F12) → Console
  3. Procure por erros em vermelho
  4. Verifique se MySQL está rodando

Problema: Erro ao cadastrar produto
Solução:
  1. Preencha todos os campos obrigatórios
  2. Verifique tamanho do nome (3-100 caracteres)
  3. Verifique conexão com backend
  4. Veja logs no terminal do backend

Problema: "Cannot connect to database"
Solução:
  1. Inicie MySQL
  2. Verifique credenciais em backend/.env
  3. Execute setup_banco.sql
  4. Atualize DB_PASSWORD em .env se necessário
